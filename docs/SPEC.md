## 1. מקורות הנתונים

כולם פתוחים, חינמיים, ללא הרשמה וללא מפתח. נשמרים בעותק מקומי (cache) לעבודה גם offline.

| מקור | מה נותן | תפקיד |
|---|---|---|
| **martj42/international_results** (GitHub, CSV) | ~47K תוצאות נבחרות מ-1872 (כולל ניטרליות מגרש) | חישוב Elo + חוזק התקפה/הגנה + Backtest |
| **openfootball/world-cup** (GitHub) | בתים, לוח משחקים ותוצאות מונדיאל 2026 | מבנה הטורניר + תוצאות |
| **Wikipedia — מונדיאל 2026** | סגלים, אימות, גיבוי | השלמה ובדיקה |
| **Elo — מחושב פנימית** | דירוג חוזק לכל נבחרת | נגזר מהתוצאות ההיסטוריות |
| **הזנה ידנית** | תוצאות חיות, פציעות/הרחקות, כל מה שחסר | מקור גיבוי תמידי |

## 2. כיצד יחושבו התחזיות

צינור בן 5 שלבים:

1. **Elo** — לכל נבחרת דירוג חוזק, מחושב מההיסטוריה ומתעדכן אחרי כל משחק.
2. **תוחלת שערים (λ)** — המרת פער ה-Elo + חוזקי התקפה/הגנה לתוחלת שערים לכל צד.
3. **Dixon-Coles (Poisson דו-משתני)** — בניית מטריצת הסתברויות לכל תוצאה אפשרית, עם תיקון לתוצאות נמוכות (0:0, 1:0, 1:1).
4. **פלט למשחק** — מהמטריצה נגזרים בו-זמנית: **P(ניצחון/תיקו/הפסד)**, **התוצאה המדויקת הסבירה ביותר**, ותוחלת השערים.
5. **סימולציית טורניר (Monte-Carlo, ~10,000 הרצות)** — סיכויי עלייה מהבית, הגעה לשלבים, וזכייה בגביע.

**אמינות:** כיול נמדד ב-Backtest על מונדיאלים קודמים (Brier-score, Log-loss) — לא "ניחוש", אלא מודל מכויל.

## 3. גורמים שיקבלו משקל בחיזוי

המשקולות **נכוילות אוטומטית ב-Backtest** ולא נקבעות שרירותית. הגורמים:

| גורם | השפעה |
|---|---|
| **חוזק בסיס (Elo)** | עיקרי — פער הדירוג בין הנבחרות |
| **התקפה / הגנה** | כמה כל נבחרת כובשת וסופגת (משוקלל לפי עדכניות) |
| **טופס אחרון** | תוצאות 5-10 המשחקים האחרונים (דעיכה מעריכית) |
| **יתרון מארחת / מגרש ניטרלי** | בונוס לארה"ב/מקסיקו/קנדה; שאר המשחקים ניטרליים |
| **חשיבות המשחק** | משחקי טורניר משפיעים על Elo יותר מידידות (K-factor) |
| **דעיכת זמן** | משחקים עדכניים שוקלים יותר מישנים |
| **זמינות סגל** | פציעות/הרחקות של שחקני מפתח (הזנה ידנית במצב חינמי) |

## 4. כיצד המערכת תלמד מתוצאות המונדיאל

לאחר כל תוצאה שנכנסת (אוטומטית מ-openfootball או בהזנה ידנית), רץ עדכון מתגלגל:

1. **עדכון Elo** — מנצחת עולה, מפסידה יורדת, משוקלל לפי הפרש השערים וחוזק היריבה.
2. **רענון טופס** — הטורניר נכנס למשקל הצורה העדכנית.
3. **חישוב טבלאות מחדש** + שוברי-שוויון פיפ"א.
4. **בניית עץ הנוקאאוט מחדש** לפי הטבלאות.
5. **הרצת Monte-Carlo מחדש** — כל סיכויי ההתקדמות מתעדכנים.
6. **מעקב כיול חי** — כל תחזית נשמרת כ-snapshot ומושווית לתוצאה בפועל; כך רואים בזמן אמת כמה המודל מדויק, ואפשר לכוונן.

זוהי למידה במובן בייסיאני: ידע מוקדם מההיסטוריה (prior), שמתעדכן לפי הראיות מהטורניר.

## 5. מסכי המערכת (Streamlit)

| מסך | תוכן |
|---|---|
| **בית / היום** | משחקי היום והקרובים עם תחזית מהירה |
| **כרטיס תחזית למשחק** | P(נצ/תיקו/הפ), תוצאה מדויקת, תוחלת שערים, ניתוח מילולי + הגורמים המשפיעים |

| **דיוק המודל** | מדדי Backtest, כיול (Brier/Log-loss), תחזית מול מציאות |

## 6. חיבור API חיצוני בעתיד — ללא שינוי מהותי

המפתח הוא **ממשק ספק אחיד (`DataProvider`)** שכל מקורות הנתונים מצייתים לו:

```
DataProvider (ממשק מופשט)
   ├─ get_fixtures()     ├─ get_results()
   ├─ get_standings()    ├─ get_squads()
   └─ get_injuries()
        │
        ├─ OpenDataProvider     ← פעיל (GitHub, חינם)
        ├─ ManualProvider       ← פעיל (הזנה ידנית)
        └─ APIFootballProvider  ← עתידי (מושבת כעת)
```

- שכבת מיזוג משלבת מקורות (פתוח + ידני), והבחירה נעשית ב-`config`.
- **הוספת API בעתיד = מחלקה אחת חדשה שמממשת את הממשק + דגל ב-config + מפתח ב-.env.**
- **אפס שינוי** במודל החיזוי, בלוגיקת הטורניר, ב-UI או במבנה ה-DB.
- הלקוח שכבר נבנה (`api_football.py`) יותאם לממשק זה — שום עבודה לא הולכת לאיבוד.

## 7. דרישות תצוגה נוספות (v1.1)

- **לכל משחק:** התוצאה הסבירה ביותר + **5 התוצאות המובילות עם הסתברות** (נשמר בעמודה `top_scores`).
- **רמת ביטחון** לכל תחזית: נמוכה / בינונית / גבוהה — נגזרת מחדות התפלגות ההסתברויות (עמודה `confidence`).
- **טבלת בתים** ו**עץ נוקאאוט** מתעדכנים אוטומטית עם כל תוצאה.
- **Dashboard ראשי:** משחקים קרובים, תחזיות קרובות, טבלת בתים, מצב הטורניר.
- **סקירה בדיעבד:** מה נחזה מול מה שקרה בפועל + אחוזי הצלחה מצטברים (פגיעה בתוצאה ובמנצח, Brier).
- **הרחבות עתידיות** (פציעות/הרחקות/סגלים/הימורים) — דרך ממשק הספקים, ללא שינוי מבני.
